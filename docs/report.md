## 1. Техническое задание

### 1.1 Назначение программы

Программа рассчитывает стоимость ремонта мобильного телефона в сервисном центре.

### 1.2 Входные данные

| Поле | Тип | Описание |
|------|-----|----------|
| Модель телефона | строка | iPhone, Samsung, Xiaomi, Google Pixel, Другое |
| Тип ремонта | строка | экран, аккумулятор, материнская плата |
| Срочность | строка | обычный, срочный, экстренный |
| Дополнительные услуги | список | выезд мастера, защитное стекло |
| Промокод | строка | SERVICE10, REMONT26 |

### 1.3 Выходные данные

- Детализированный чек
- Итоговая сумма
- Рекомендация (оригинал или аналог)

### 1.4 Функциональные требования

1. Расчёт стоимости работы по модели и типу ремонта
2. Автоматический подбор стоимости запчастей из базы
3. Добавление платы за диагностику (500 руб, кроме материнской платы)
4. Применение коэффициента срочности (1.0, 1.3, 1.5)
5. Применение скидки по промокоду
6. Сравнение оригинальной и аналоговой запчасти
7. Логирование всех операций в файл

## 2. Модульная структура

### 2.1 Модули

| Модуль | Файл | Функции | Назначение |
|--------|------|---------|------------|
| Точка входа | main.py | main() | Запуск программы |
| Ввод данных | input_handler.py | get_model(), get_repair(), get_parts(), get_urgency(), get_services(), get_promocode(), get_all_data() | Общение с пользователем |
| Расчёты | calculator.py | calc_labor(), calc_total(), compare_options() | Бизнес-логика |
| Конфигурация | config.py | PRICES, URGENCY, PROMOCODES | Хранение цен |
| Логирование | logger.py | write_log(), log_start(), log_input(), log_result(), log_error() | Запись в файл |
| Чек | receipt.py | print_receipt() | Вывод на экран |
| Тесты | test_*.py | test_*() | Модульное тестирование |

### 2.2 Схема передачи данных
input_handler.py
↓ (словарь data)
main.py
↓ (словарь data)
calculator.py
↓ (словарь result)
main.py
↓ (data + result)
receipt.py → консоль
main.py
↓ (data)
calculator.py (compare_options) → рекомендация
